# Web App Screenshots

## How to Generate Screenshots

Since this was developed in a CLI environment, actual screenshots need to be generated by opening the web app in a browser.

### Steps to Create Screenshots:

1. **Open the Web App:**
   ```bash
   cd /path/to/todo-app-demo
   open public/index.html
   ```

2. **Add Sample Tasks:**
   - Add a high-priority work task: "Complete project report"
   - Add a medium-priority study task: "Learn JavaScript"
   - Add a low-priority shopping task: "Buy groceries"
   - Mark one task as completed

3. **Screenshot Light Mode:**
   - Ensure light mode is active
   - Take a full-page screenshot
   - Save as `screenshots/web-light-mode.png`

## What the App Looks Like

### Light Mode Features:
- Clean white background (#f5f7fa)
- Blue accent color (#4a90e2)
- Colorful priority badges (red/orange/green)
- Smooth shadows and rounded corners
- Professional, modern appearance

### Dark Mode Features:
- Dark background (#1a1a2e)
- Darker card backgrounds (#16213e)
- Adjusted text colors for readability
- Same colorful priority badges
- Easy on the eyes for night use

### UI Elements:
- **Header**: App title, stats summary, theme toggle
- **Add Task Form**: Input field, priority dropdown, category dropdown, add button
- **Controls**: Filter buttons (All, Work, Personal, Study, Shopping), Sort buttons
- **Task List**: Draggable cards with checkboxes, priority/category badges, edit/delete buttons
- **Stats Panel**: Category and priority statistics, collapsible

### Interactions:
- Drag and drop to reorder tasks
- Double-click task text to edit inline
- Hover effects on buttons and cards
- Smooth transitions and animations
- Keyboard shortcuts work seamlessly

## Expected Appearance

The web app should look similar to modern todo apps like Todoist or Microsoft To Do, with:
- Card-based task items
- Color-coded priorities
- Category tags
- Clean, minimalist design
- Responsive layout that works on all screen sizes
